# Hermes Studio UI Redesign Spec (v1)

この文書は全UIリニューアルの唯一のデザイン基準。実装エージェントは必ずこの仕様に従うこと。

## コンセプト

- Codex app のようなシンプルで静かなUI。余計な文字を排し、アイコン中心。
- 説明文はすべて InfoTip（`components/info-tip.tsx`）のツールチップに退避する。
- ただし詳細設定への導線（設定アイコン→設定サーフェス、プロファイル右クリック/長押し等）は必ず残す。
- トーンは起動時スプラッシュ（`device-login.tsx` のWebGL "bright lab" シェーダー）に統一:
  明るいペーパーホワイト基調 + ミント/アクアのアクセント + コーラル(エラー) + アンバー(警告)。
- 現行のダーク "night blueprint" テーマは既定から外し、スプラッシュ準拠のライトテーマを既定にする。
  (appearance のテーマ切替機構は壊さない。`midnight` はダーク派向けに残してよいが、既定は新ライト。)

## デザイントークン（styles.css の `:root` を置き換え。packages/ui-tokens も同期）

```css
--bg: #F4F7F9;            /* シェーダー c1/c3 由来の空気感 */
--surface: #FFFFFF;
--surface-2: #EDF3F4;     /* シェーダー c2 ミントがかった面 */
--ink: #1C2733;           /* 本文テキスト */
--ink-muted: rgba(28,39,51,.55);
--accent: #17A98C;        /* terminal aqua を可読コントラストに調整 */
--accent-soft: rgba(85,214,190,.16);
--coral: #E85B48;         /* destructive / error */
--amber: #E0A93E;         /* waiting / warning */
--line: rgba(28,39,51,.12);
--radius-sm: 8px; --radius-md: 12px; --radius-lg: 16px;
--text-xs: 12px; --text-sm: 13px; --text-md: 14px; --text-lg: 16px;  /* ×var(--font-scale) は維持 */
--font-sans: "Noto Sans JP", system-ui, sans-serif;  /* display も同系で統一 */
--shadow-sm: 0 1px 2px rgba(20,35,45,.06);
--shadow-md: 0 8px 24px rgba(20,35,45,.10);
--shadow-lg: 0 24px 56px rgba(20,35,45,.14);
```

ルール:

- ボーダーは常に `1px solid var(--line)`。太さ・色のバリエーションを作らない（選択状態のみ `--accent`）。
- 角丸は sm/md/lg の3種のみ。ピル形状はアバター/ステータスドットのみ許可。
- ボタン高さ: 36px(標準) / 32px(コンパクト) / 44px(SPタッチターゲット最小)。
- 余白スケール: 4/8/12/16/24/32 のみ使用。
- 文字は最小12px。見出しはページ 20px / パネル 16px、weight 700 まで（900は使わない）。
- 影は3段のみ。ホバーは背景 `--surface-2` への変化を基本とする。

## 文字削減・ツールチップ方針

- ラベル+説明文が並ぶ箇所は「アイコンまたは短いラベル + InfoTip」に変換。
- ナビ・アクション類はアイコン化し `aria-label` + `title` を必ず付ける。
- ステータスは色ドット+ツールチップ。文言バッジは廃止。
- i18n (`i18n.ts`) のキーは削除せず、ツールチップ側で利用する。

## レスポンシブ

- PC: サイドレール(アイコンのみ56px) + メインステージ + 右ワークスペースドロワー。ホバー操作前提。
- SP(<=768px): 下部タブバー(アイコンのみ)。ホバー不可のため InfoTip はタップで開閉。
  タッチターゲット44px以上。モーダルはフルスクリーンシート(上角丸 lg)。
- 既存の mobile-routes / use-mobile-overlay の仕組みは維持して見た目のみ刷新。

## 3Dスタジオ（office-3d.tsx）

- 「マザー2風の温かみあるドット絵 × 洗練されたモダン」なスタジオ表現に刷新。
  - テクスチャはすべて canvas 生成のピクセルアート（NearestFilter、低解像度→拡大）。
  - 床は暖色チェッカー/木目調ドットタイル、壁は淡いクリーム、家具は彩度低めの暖色ドット絵。
  - ライティングは柔らかく明るく（スプラッシュのbright labトーン）。暗い舞台照明はやめる。
  - キャラは既存アバターsprite を維持しつつ、ドット絵風の縁取り・影(楕円ドット)で馴染ませる。
- 視認性最優先: コントラスト確保、ホバー対象は明確なハイライト、名札は小さく読みやすく。
- 固定位置要件: タスクボード(kanbanオーバーレイの入口となる3D内ボード) と default プロファイルの席は
  常に固定位置（defaultは現行どおり desk 0 / 受付位置、ボードは入口付近の壁）に配置する。

## 変更禁止事項

- 挙動・API・ストア・ルーティングのロジックは変更しない（見た目とマークアップのみ）。
- テスト対象の純ロジック(ts)ファイルは変更しない。
- ローカルでの build/test/lint は実行しない（CI が検証源）。typecheck も不要。
